# Architecture

**Analysis Date:** 2026-02-28

## Pattern Overview

**Overall:** Multi-system content pipeline + SPA reader

The project is a content-generation platform with three distinct subsystems:

1. **Python automation pipeline** (`bot/`) ‚Äî discovers, writes, edits, and publishes blog posts
2. **Node.js Express API** (`server.js`) ‚Äî CRUD on markdown posts for editors
3. **React SPA** (`src/`) ‚Äî reads and displays blog content

**Key Characteristics:**
- File-system as the integration layer ‚Äî markdown files in `src/blog/posts/` and JSON in `public/` are the shared contract
- Dual data path: frontend reads from Express API (dev) or static JSON (production/fallback)
- Bot pipeline is autonomous ‚Äî scrape, generate, publish, git push ‚Äî runs on a 30-minute cron
- No shared runtime between Python bot and Node.js/React ‚Äî only shared files

## Layers

**Frontend (React SPA):**
- Purpose: Landing page, blog listing, article reader, editors
- Location: `src/`
- Contains: Components, blog pages, utils, data modules, types
- Depends on: `react-router-dom`, `blogUtils`, `public/*.json`
- Used by: End users, editors

**API (Express):**
- Purpose: CRUD for blog posts; serves markdown as JSON
- Location: `server.js` (root)
- Contains: REST endpoints, auth middleware, rate limiting
- Depends on: `gray-matter`, `fs`, `src/blog/posts/`
- Used by: Frontend editors (LocalEditor, AdvancedBlogEditor)

**Bot Pipeline (Python):**
- Purpose: Scrape sources, generate articles via AI, publish to repo
- Location: `bot/`
- Contains: scraper, ai_writer, ai_refiner, ai_editor, amazon_linker, image_fetcher, publisher
- Depends on: `config.py`, `directives/`, external APIs (Reddit, RSS, OpenRouter, Amazon, Unsplash)
- Used by: Cron / Task Scheduler

**Build / Scripts:**
- Purpose: Markdown ‚Üí JSON, Vite build, Netlify 404 fix
- Location: `scripts/`
- Contains: `build-blog.js`, `debug-netlify-404.js`
- Depends on: `gray-matter`, `src/blog/posts/`
- Used by: `npm run build`, `server.js` (post-save), `bot/publisher.py`

## Data Flow

**Content pipeline (Reddit/RSS ‚Üí published blog):**

1. `bot/scraper.py` ‚Äî Pulls stories from Reddit and RSS feeds. Scores by relevance, deduplicates via `seen_posts.db`, returns top N `StoryBrief` dicts.
2. `bot/ai_writer.py` ‚Äî Sends brief to AI, returns draft with frontmatter.
3. `bot/ai_refiner.py` ‚Äî Enriches draft with concrete details (product names, prices).
4. `bot/ai_editor.py` ‚Äî Editorial pass, internal linking, SEO.
5. `bot/amazon_linker.py` ‚Äî Matches article to `affiliate-products.json`, adds Amazon links.
6. `bot/image_fetcher.py` ‚Äî Fetches cover + article images (Amazon, Unsplash, stock).
7. `bot/publisher.py` ‚Äî Writes `src/blog/posts/<slug>.md`, runs `npm run build:blog`, git push, Discord notify.

**Frontend data flow:**

1. `blogUtils.ts` ‚Äî `getAllPosts()` / `getPostBySlug()` try API first (`VITE_API_URL` or `http://localhost:3001/api`), fall back to `/blog-posts.json`.
2. In-memory cache (`cachedPosts`, `cachedTags`) for instant display via `getAllPostsSync()`.
3. `BlogHome` ‚Äî Fetches posts, filters by tag/search, paginates.
4. `BlogPost` ‚Äî Fetches single post, related products (`productData.ts` ‚Üí `affiliate-products.json`), related posts (topic scoring).

**State Management:**
- No global state library. Each page uses `useState`/`useEffect`. `blogUtils` maintains a module-level cache for posts and tags.

## Key Abstractions

**StoryBrief (bot):**
- Purpose: Story metadata from scraper
- Location: `bot/scraper.py`
- Shape: `{id, title, summary, source_url, source_type, raw_content, relevance_score, topic, theme, story_angle, personal_score}`

**Draft (bot):**
- Purpose: Article in progress through pipeline stages
- Location: Passed between `ai_writer`, `ai_refiner`, `ai_editor`, `amazon_linker`, `image_fetcher`, `publisher`
- Shape: `{slug, frontmatter: {...}, content: str, amazon_search_queries?, featured_product_keywords?}`

**BlogPost (frontend):**
- Purpose: Blog post type
- Location: `src/types.ts`
- Shape: `{id, slug, title, excerpt, content, publishedDate, author, tags, readingTime, coverImage, contentBlocks, images, amazonProducts, featuredProductId, recommendedProductIds, autoGenerated, sourceUrl, relatedPostSlugs}`

**ContentBlock (frontend):**
- Purpose: Block-based content for editors and renderers
- Location: `src/types.ts`
- Types: `title`, `headline`, `excerpt`, `divider`, `text`, `image`, `product`, `snippet`

**Product / AmazonProduct (frontend):**
- Purpose: Affiliate product types
- Location: `src/types.ts`
- `Product`: From `affiliate-products.json` (curated list)
- `AmazonProduct`: From frontmatter (bot-generated)

## Entry Points

**Frontend:**
- `index.html` ‚Äî Loads `/src/main.tsx`
- `src/main.tsx` ‚Äî React bootstrap, `createBrowserRouter`, mounts to `#root`
- `src/App.tsx` ‚Äî Landing page (no Layout wrapper)

**Server:**
- `server.js` ‚Äî Express app, `node server.js` or `npm run server`, port 3001

**Bot:**
- `bot/scheduler.py` ‚Äî Pipeline orchestrator, `python bot/scheduler.py --once` or `--daemon`

**Build:**
- `scripts/build-blog.js` ‚Äî `npm run build:blog`, markdown ‚Üí `public/blog-posts.json`
- `vite build` ‚Äî Production bundle

**Dev:**
- `npm run dev:full` ‚Äî Runs `server` + `vite` concurrently

## Error Handling

**Strategy:** Per-layer try/catch; no global error boundary for routing.

**Bot:** Retries via tenacity (2‚Äì3 attempts). Writer/refiner abort on content below min words. On crash, partial drafts saved to `bot/.tmp/drafts/` for resume. Discord webhook on success/failure.

**Frontend:** `blogUtils` catches fetch errors, returns `[]` or `null`. Pages show loading/error states. `ErrorBoundary` wraps blog routes in `main.tsx`.

**Server:** try/catch per endpoint, returns 400/404/500 JSON. `build:blog` failures logged but do not block API response.

## Cross-Cutting Concerns

**Logging:** Bot uses Python `logging` with console + rotating file handlers in `bot/logs/`. Frontend uses `console.log`/`console.error`.

**Validation:** Slug validation via `SLUG_PATTERN` in `server.js`. Frontend relies on API responses.

**Authentication:** Express `requireAuth` checks `Authorization: Bearer <BLOG_API_KEY>` for POST/DELETE. Frontend `blogUtils.savePost`/`deletePost` do not currently send this header ‚Äî editors will receive 403 unless configured.

**Configuration:** Bot uses `bot/config.py` (dataclasses, `bot/.env`). Server uses `BLOG_API_KEY`, `PORT` from env.
