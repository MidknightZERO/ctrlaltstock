## Directory Purposes

**`src/`:**
- Purpose: React SPA source
- Contains: Components, pages, blog feature, types, styles
- Key files: `main.tsx`, `App.tsx`, `types.ts`

**`src/blog/`:**
- Purpose: Blog listing, article reader, editors
- Contains: BlogHome, BlogPost, LocalEditor, AdvancedBlogEditor, components, data, utils
- Key files: `BlogHome.tsx`, `BlogPost.tsx`, `AdvancedBlogEditor.tsx`, `LocalEditor.tsx`, `utils/blogUtils.ts`

**`src/blog/posts/`:**
- Purpose: Markdown source of truth for blog posts
- Contains: One `.md` file per post (slug as filename)
- Written by: `server.js` (editors), `bot/publisher.py` (bot)

**`src/blog/data/`:**
- Purpose: Static data, pillar posts, product matching, tag hierarchy
- Contains: `blogPosts.ts`, `pillarPosts.ts`, `productData.ts`, `tagHierarchy.ts`, `stockImages.ts`

**`bot/`:**
- Purpose: Content pipeline (scrape → write → refine → edit → images → publish)
- Contains: Python scripts, directives, config, logs, SQLite DB
- Key files: `scheduler.py`, `scraper.py`, `publisher.py`, `config.py`

**`public/`:**
- Purpose: Static assets and generated JSON consumed by frontend
- Contains: `blog-posts.json`, `affiliate-products.json`, `blog-categories.json`, Logo, `_redirects`

**`scripts/`:**
- Purpose: Build and utility scripts
- Contains: `build-blog.js`, `debug-netlify-404.js`

**`dist/`:**
- Purpose: Vite production build output
- Generated: Yes
- Committed: Typically no (deploy artifact)

## Key File Locations

**Entry Points:**
- `index.html`: HTML shell, loads `/src/main.tsx`
- `src/main.tsx`: React bootstrap, router, ToastProvider
- `server.js`: Express API server
- `bot/scheduler.py`: Pipeline entry

**Configuration:**
- `vite.config.ts`: Vite config
- `tailwind.config.js`: Tailwind
- `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json`: TypeScript
- `bot/config.py`: Bot config (AI, Reddit, RSS, Amazon, Git, etc.)
- `bot/.env`: Bot env vars (API keys)

**Core Logic:**
- `src/blog/utils/blogUtils.ts`: Data fetching (API + static fallback), cache, save/delete
- `src/blog/data/productData.ts`: Affiliate product fetch, tag-based recommendations
- `scripts/build-blog.js`: Markdown parsing, JSON generation

**Routing:**
- `src/main.tsx`: `createBrowserRouter` with flat routes

**Testing:**
- No test directory detected

## Naming Conventions

**Files:**
- React components: PascalCase (e.g. `BlogHome.tsx`, `BlockRenderer.tsx`)
- Utilities/data: camelCase (e.g. `blogUtils.ts`, `productData.ts`)
- Bot scripts: snake_case (e.g. `ai_writer.py`, `amazon_linker.py`)

**Directories:**
- `src/blog/components/`: React components for blog
- `src/blog/data/`: Data modules
- `bot/directives/`: Markdown SOPs

**Routes:**
- `/blog` — listing
- `/blog/:slug` — article
- `/blog/editor` — LocalEditor
- `/blog-editor`, `/blog-editor/:slug` — AdvancedBlogEditor

## Where to Add New Code

**New blog component:**
- Implementation: `src/blog/components/<Name>.tsx`
- Used by: BlogHome, BlogPost, or editors

**New data module:**
- Implementation: `src/blog/data/<name>.ts`
- Export types from `src/types.ts` if needed

**New bot pipeline step:**
- Implementation: `bot/<step_name>.py`
- Wire into `bot/scheduler.py` in `run_pipeline_for_story`

**New page/route:**
- Add route in `src/main.tsx` `createBrowserRouter`
- Create component in `src/components/` or `src/blog/`
- Wrap with `Layout` if using shared header/footer

**Utilities:**
- Shared helpers: `src/blog/utils/` or `src/` root
- Bot helpers: `bot/utils.py`

## Special Directories

**`bot/logs/`:**
- Purpose: Rotating logs (scheduler, publish, scraper, etc.)
- Generated: Yes (by Python logging)
- Committed: Typically no

**`bot/.tmp/drafts/`:**
- Purpose: Partial drafts for resume after pipeline crash
- Generated: Yes
- Committed: No

**`dist/`:**
- Purpose: Vite build output
- Generated: Yes
- Committed: No (deploy from CI)

**`public/`:**
- Purpose: Static assets; `blog-posts.json` is generated, others may be manual
- Generated: Partially (`blog-posts.json` by build:blog)
- Committed: Yes
