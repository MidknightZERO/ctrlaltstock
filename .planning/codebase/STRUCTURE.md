## Directory Purposes

**bot/**  
- Purpose: Full automation for the blog: scrape → write → refine → edit → link → images → publish, plus backfill and fix-existing workflow.  
- Contains: Python scripts, config, directives, .tmp, logs, stock_images.json, used_images.json, seen_posts.db.  
- Key files: `scheduler.py`, `scraper.py`, `backfill_content.py`, `image_fetcher.py`, `publisher.py`, `validate_existing_content.py`, `generate_fix_list.py`, `run-fix-existing.ps1`, `config.py`.

**bot/directives/**  
- Purpose: Markdown SOPs for scraping and fact-checking.  
- Contains: `scrape.md`, `fact_check.md`.  
- Note: `scrape.md` refers to `execution/scraper.py`; the actual script is `bot/scraper.py`. There is no `execution/` directory at repo root.

**bot/.tmp/**  
- Purpose: Intermediate outputs for resume and backfill.  
- Contains: `drafts/` (partial pipeline drafts), `validation-report.json` (from validate_existing_content), `fix-list.json` (from generate_fix_list).  
- Generated: Yes. Not committed (should be in .gitignore).  
- Backfill and image refresh depend on validation-report and fix-list being produced first (see run-fix-existing.ps1 order).

**execution/**  
- Not present. All “execution” scripts live under `bot/`. AGENTS.md describes a 3-layer layout with `execution/`; in this repo that role is fulfilled by `bot/*.py`.

**scripts/**  
- Purpose: Node build tooling.  
- Key file: `build-blog.js` — reads `src/blog/posts/*.md`, writes `public/blog-posts.json` (and categories). Invoked by `npm run build:blog` and by `bot/publisher.py` after writing a new post.

**src/blog/posts/**  
- Purpose: Canonical markdown posts. One file per post: `<slug>.md`.  
- Written by: `bot/publisher.py` (new posts), `bot/backfill_content.py` (edits in place).  
- Consumed by: `scripts/build-blog.js` to generate `public/blog-posts.json`.

**public/**  
- Purpose: Static assets and generated JSON for the app.  
- Key: `blog-posts.json` (generated), `affiliate-products.json` (curated products; bot can fill imageUrl via fix_affiliate_images.py).

## Key File Locations

**Entry points**
- `bot/scheduler.py`: Main cron entry (e.g. `python bot/scheduler.py --once`).
- `bot/run-fix-existing.ps1`: Fix-existing workflow (validate → fix list → backfill → backfill --images-only → build:blog).
- `bot/resume_draft.py`: Resume a partial draft after pipeline crash.
- `scripts/build-blog.js`: Regenerate blog JSON from markdown.

**Configuration**
- `bot/config.py`: Central config (paths, API keys from env). Loads `bot/.env`.
- `bot/.env`: Secrets (OPENROUTER, REDDIT, UNSPLASH, PEXELS, AMAZON, etc.).

**Backfill, image fetch, content quality**
- `bot/backfill_content.py`: Tags, internal/Amazon links, excerpts, inline images, featured product; `--images-only` for cover refresh. Reads `bot/.tmp/validation-report.json` and `bot/.tmp/fix-list.json` when present.
- `bot/validate_existing_content.py`: Writes validation-report.json; optional --fix for H1 and partial-word links.
- `bot/generate_fix_list.py`: AI batch to build fix-list.json (image_search_queries per slug) for backfill --images-only.
- `bot/image_fetcher.py`: Assigns coverImage + images per draft; uses stock_images.json, used_images.json, and 7-day cover exclusion from blog-posts.json.
- `bot/fix_affiliate_images.py**: Fills imageUrl in public/affiliate-products.json (standalone; not part of backfill).

**Content deduplication**
- `bot/scraper.py`: Source-level dedup via seen_posts DB and similarity vs existing posts (title+tags, last 7 days). No body/content dedup.

**Publishing**
- `bot/publisher.py`: Writes .md, runs build:blog, git add/commit/push, build hook, Discord.

## Naming Conventions

**Files**
- Bot scripts: `snake_case.py` (e.g. `backfill_content.py`, `image_fetcher.py`).
- Posts: `<slug>.md` where slug is URL-safe (e.g. `amd-adrenalin-26-2-2-rx-7000-stability-update.md`).
- Config/data: `kebab-case.json` or `snake_case.json` (e.g. `blog-posts.json`, `stock_images.json`, `fix-list.json`).

**Directories**
- `bot/`, `src/`, `public/`, `scripts/`, `.planning/` — lowercase.  
- `bot/.tmp/` for intermediates; `bot/logs/` for log files.

## Where to Add New Code

**New pipeline step**
- Add a new `bot/<step>.py` and call it from `bot/scheduler.py` in the correct order (see ARCHITECTURE.md). Consider whether it should run before or after amazon_linker/image_fetcher/publisher.

**New backfill behavior**
- Extend `bot/backfill_content.py` (e.g. new fix function and CLI flag). If it needs prior analysis, add a script that writes to `bot/.tmp/` and document the order in run-fix-existing.ps1 or ARCHITECTURE.md.

**New directive**
- Add `bot/directives/<topic>.md`. Point to the actual script path under `bot/` (not `execution/`).

**New frontend page/component**
- `src/` under the appropriate feature (e.g. `src/blog/components/`). Blog data comes from `public/blog-posts.json` produced by `scripts/build-blog.js`.

## Scripts That Do Backfill, Image Fetch, or Content Deduplication

| Script | Role | Depends on |
|--------|------|------------|
| `bot/backfill_content.py` | Backfill tags, links, excerpts, inline images, featured product; `--images-only` refreshes covers | validation-report.json (optional), fix-list.json for --images-only |
| `bot/validate_existing_content.py` | Audit/fix H1 and partial-word links; write validation-report.json | — |
| `bot/generate_fix_list.py` | AI batch → fix-list.json (image_search_queries per slug) | — |
| `bot/image_fetcher.py` | Per-article cover + images (new posts and when backfill calls fetch_images) | stock_images.json, used_images.json, blog-posts.json (recent covers) |
| `bot/fix_affiliate_images.py` | Populate imageUrl in affiliate-products.json | — |
| `bot/scraper.py` | Source dedup (seen_posts, similarity vs existing titles/tags); no content/body dedup | blog-posts.json, seen_posts.db |

**Execution order for fix-existing (run-fix-existing.ps1):**  
1) validate_existing_content → 2) generate_fix_list → 3) backfill_content → 4) backfill_content --images-only → 5) npm run build:blog.
