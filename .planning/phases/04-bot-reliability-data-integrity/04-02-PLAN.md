  <name>Task 2: Make used_images.json writes atomic</name>
  <files>bot/image_fetcher.py</files>
  <action>
Modify `_save_used_images()` (lines 96-104) to use atomic write pattern: write to a temp file in the same directory, then `os.replace()` to the final path.

Add `import os` at the top if not already present (check — it's not currently imported).

Replace the current implementation:
```python
def _save_used_images(urls: List[str]) -> None:
    """Persist recently used image URLs (atomic write)."""
    try:
        tmp_path = USED_IMAGES_PATH.with_suffix(".tmp")
        tmp_path.write_text(
            json.dumps({"urls": urls[:USED_IMAGES_MAX]}, indent=2),
            encoding="utf-8",
        )
        os.replace(str(tmp_path), str(USED_IMAGES_PATH))
    except Exception as e:
        log.warning("Could not save used_images.json: %s", e)
        try:
            tmp_path.unlink(missing_ok=True)
        except Exception:
            pass
```

Key points:
- `os.replace()` is atomic on the same filesystem (POSIX guarantee, Windows best-effort but still much safer than direct write)
- Write to `.tmp` suffix in same directory ensures same filesystem
- Clean up temp file on failure so it doesn't linger
- `with_suffix(".tmp")` creates `used_images.tmp` in the same directory

In `scripts/build-blog.js`, the `buildBlog()` function (line 211) currently does:
```js
fs.writeFileSync(OUTPUT_FILE, JSON.stringify(posts, null, 2));
```

Replace with atomic write pattern:
```js
const tmpFile = OUTPUT_FILE + '.tmp';
fs.writeFileSync(tmpFile, JSON.stringify(posts, null, 2));
fs.renameSync(tmpFile, OUTPUT_FILE);
```

`fs.renameSync()` is atomic on the same filesystem (same as POSIX rename). Since both files are in the `public/` directory, they're on the same filesystem.

No try/catch needed here since the outer try/catch in the script already handles errors and calls `process.exit(1)`. But do add cleanup of the temp file in the catch:
```js
try {
  buildBlog();
  console.log('Blog build completed successfully!');
} catch (error) {
  console.error('Blog build failed:', error);
  try { fs.unlinkSync(OUTPUT_FILE + '.tmp'); } catch {}
  process.exit(1);
}
```
