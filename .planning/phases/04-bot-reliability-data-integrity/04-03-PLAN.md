Replace all plain FileHandler instances with RotatingFileHandler across the entire bot pipeline to prevent unbounded log growth.

Purpose: Bot logs grow without bound. On a cron running every 15-30 minutes, log files can reach gigabytes within weeks. RotatingFileHandler caps each file at 10MB with 3 backups (40MB max per module).
Output: All 8 bot modules use RotatingFileHandler(maxBytes=10MB, backupCount=3).
</objective>

  <action>
Apply the same mechanical change to all 8 bot modules listed above.

**For each file:**

1. Add the import at the top (after the existing `import logging` line):
```python
from logging.handlers import RotatingFileHandler
```

2. In the `logging.basicConfig(handlers=[...])` block, replace:
```python
logging.FileHandler(Path(config.bot.logs_dir) / "LOGNAME.log", encoding="utf-8"),
```
with:
```python
RotatingFileHandler(
    Path(config.bot.logs_dir) / "LOGNAME.log",
    maxBytes=10 * 1024 * 1024,
    backupCount=3,
    encoding="utf-8",
),
```

**File-specific log names (preserve existing names):**
| File | Log name |
|------|----------|
| scheduler.py | scheduler.log |
| scraper.py | scraper.log |
| publisher.py | publish.log |
| ai_editor.py | ai_editor.log |
| ai_refiner.py | ai_refiner.log |
| backfill_content.py | backfill.log |
| resume_draft.py | resume_draft.log |
| reprocess_rejected.py | reprocess_rejected.log |

**Important notes:**
- Do NOT change the StreamHandler — keep both handlers (console + rotating file)
- Do NOT change the log format string
- Do NOT change the logging level
- `RotatingFileHandler` is in `logging.handlers`, not `logging` — the import is required
- maxBytes=10*1024*1024 = 10MB, backupCount=3 = keep .log, .log.1, .log.2, .log.3
  </action>
