In scripts/build-blog.js: When setting coverImage from frontMatter, if the value looks like a srcset (contains comma and spaces, e.g. "url1 208w, url2 416w"), normalize to a single URL: take the first URL from the string (split by comma, take first segment, trim, then strip any trailing " 123w" or similar). Do not assume coverImage is always a single URL from legacy data until build is fixed; after build normalization, new builds will have single URLs.
3. Add onError on cover img elements where missing so broken URLs fall back to placeholder or hide image (avoid broken-image icon).
  </action>
