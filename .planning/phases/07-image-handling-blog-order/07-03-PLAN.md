# Phase 07-03: Image Workflow + Backfill Order

**Phase:** 07-image-handling-blog-order (plan 03)  
**Focus:** Ensure topic-relevant images (Pexels/Unsplash) and correct script execution order.

---

## Objective

1. **Images:** Use AI-generated, article-relevant search terms when fetching from Pexels/Unsplash so the same image is not reused across unrelated articles and covers match topic (no GPU image on CPU/corporate articles).
2. **Order:** Enforce and document the correct sequence: validate → generate fix list → backfill → backfill --images-only → build, and ensure cron/fix-existing scripts run in that order.

---

## Root Causes (from codebase map)

| Issue | Cause |
|-------|--------|
| Same image everywhere | Anti-reuse limited to last 50 URLs + 7-day covers; query cleaning collapses many terms to one phrase; small stock pool. |
| Irrelevant images | Fallback when AI omits `image_search_queries`; `_clean_query_for_images` overrides with primary_topic; backfill uses title-only when fix list missing. |
| Wrong/missing AI search terms | `generate_fix_list.py` not run before `backfill_content.py --images-only`; scheduler never runs fix list or image refresh. |
| Script order | No enforcement; backfill proceeds with empty validation/fix list if files missing; no warning when fix list absent for --images-only. |

---

## Execution Order (canonical)

```
1. validate_existing_content.py   → bot/.tmp/validation-report.json
2. generate_fix_list.py           → bot/.tmp/fix-list.json (AI per-post image_search_queries)
3. backfill_content.py            → tags, links, excerpts, inline images (uses validation-report)
4. backfill_content.py --images-only → refresh covers (uses fix-list.json; Pexels)
5. npm run build:blog             → public/blog-posts.json
```

**Dependencies:** Step 3 needs (1). Step 4 needs (2). Step 5 after any step that changes posts.

---

## Tasks

### 1. Tighten `bot/run-fix-existing.ps1`

- **Requirement:** Script must enforce order and fail fast when a required step fails or a required artifact is missing.
- **Actions:**
  - Set `$ErrorActionPreference = "Stop"` for critical steps (or check `$LASTEXITCODE` after each Python call and exit if non-zero).
  - After step 1: verify `bot/.tmp/validation-report.json` exists; warn if missing (step 3 will still run but with empty report).
  - After step 2: verify `bot/.tmp/fix-list.json` exists; **if missing, abort before step 4** with a clear message: "Fix list missing. Run generate_fix_list.py first. Aborting image refresh."
  - Before step 4: if fix-list still missing, do not run `backfill_content.py --images-only` (or run only if user passed a "force" flag).
  - Document at top of script: "Step order is required. Do not skip 1 or 2 if you want topic-relevant images and distribution-aware linking."
- **Verify:** Run script; after step 2 delete fix-list.json and run again — script should abort before step 4 with clear message.

### 2. Backfill: require or warn when fix list missing for `--images-only`

- **Requirement:** When `backfill_content.py --images-only` is run without `bot/.tmp/fix-list.json`, either fail or warn loudly so users know images will be title-based only.
- **Actions:**
  - In `backfill_content.py`, when `args.images_only` and `load_fix_list()` returns empty or missing file:
    - Log: `WARNING: fix-list.json missing or empty. Run generate_fix_list.py first for topic-relevant images. Proceeding with title-based search only.`
  - Add optional flag `--require-fix-list`: if set and fix list missing/empty, log error and exit with code 1. (Default: warn and continue for backward compatibility.)
- **Verify:** Run `python bot/backfill_content.py --images-only` with fix-list removed → warning in log. Run with `--require-fix-list` and no fix list → exit 1.

### 3. Scheduler: optional image refresh after backfill

- **Requirement:** Cron can optionally run fix-list generation and cover image refresh after the usual backfill so existing posts get topic-relevant images without a separate manual run.
- **Actions:**
  - Add config (e.g. env `BOT_RUN_IMAGE_REFRESH_AFTER_CRON=1`). In `config.py` BotConfig: `run_image_refresh_after_cron: bool` from env.
  - In `scheduler.py`, after successful `run_backfill(tags=False, links=True, ...)`:
    - If `config.bot.run_image_refresh_after_cron` is True:
      - Run `generate_fix_list.main()` (or subprocess) to refresh `fix-list.json`.
      - Call `run_backfill(..., images_only=True)`.
      - Log "Image refresh (fix list + covers) completed."
  - Document in `bot/CRON-SETUP.md`: "To refresh cover images for existing posts on each run, set `BOT_RUN_IMAGE_REFRESH_AFTER_CRON=1` in bot/.env. This runs AI fix-list generation then backfill --images-only (uses more API calls)."
- **Verify:** Set env, run scheduler --once with at least one publish; logs show fix list run and image refresh. Unset env, run again; no image refresh step.

### 4. Document dependency graph in one place

- **Requirement:** Single source of truth for "fix existing" order and what each artifact is for.
- **Actions:**
  - Add or update a section in `bot/CRON-SETUP.md` (or `bot/README.md` / `directives/`): "Fix existing posts (manual): run `bot/run-fix-existing.ps1`. Order: validate → generate_fix_list → backfill → backfill --images-only → build:blog. validation-report.json enables distribution-aware linking; fix-list.json enables topic-relevant cover images for --images-only."
  - In `run-fix-existing.ps1` header comment, keep the same order and add: "Required artifacts: .tmp/validation-report.json (step 1), .tmp/fix-list.json (step 2). Step 4 will abort if fix-list is missing."
- **Verify:** Read CRON-SETUP (or chosen doc); dependency graph and order are clear.

## Success criteria

- [ ] `run-fix-existing.ps1` checks for fix-list before step 4 and aborts with clear message if missing.
- [ ] `backfill_content.py --images-only` warns when fix list missing; `--require-fix-list` exits 1 when missing.
- [ ] Scheduler optionally runs generate_fix_list + backfill --images-only when `BOT_RUN_IMAGE_REFRESH_AFTER_CRON=1`.
- [ ] Dependency graph and script order documented in CRON-SETUP (or bot docs).
